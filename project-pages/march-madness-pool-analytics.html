March Madness Pool Analytics

View on GitHub

Description

Rather than attempting to predict a perfect bracket (odds: 1 in 9.2 quintillion), this platform is built around pool-style March Madness gameplay. In these pools, players select 8 teams from the tournament field through an auction system, aiming to assemble the roster with the most total wins. The platform provides probability distributions showing how many games (0–6) each team is likely to win, helping players make strategic, data-driven decisions.

The application combines machine learning predictions with a modern full-stack web interface, including live model evaluation that tracks predictions against actual tournament results as the tournament unfolds.

Key Features

  • Win probability distributions (0–6 wins) for every tournament team
  • Historical team comparisons using ChromaDB vector similarity search
  • Power rankings and head-to-head matchup predictions
  • Pool team builder for auction-style draft preparation
  • Live results tracking with model evaluation metrics updated during the tournament
  • CI/CD pipeline with automated testing, security scanning, and linting

Technologies Used

  • Python / FastAPI
  • React / Vite
  • ChromaDB (vector similarity search)
  • Docker / Docker Compose / Nginx
